001. Привет! В этом уроке я расскажу тебе про то, как выполнить произвольный код в сценарии. Давай откроем второй экземпляр Ванессы в режиме обучения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00000_63902995945312_КакВСценарииВыполнитьКодНаЯзыке1С.png)

002. Важный момент. Помещение кода на встроенном языке 1С внутрь сценария приводит к ухудшению читаемости сценария.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00001_63902995951893_КакВСценарииВыполнитьКодНаЯзыке1С.png)

003. Эту возможность нужно использовать только в исключительных случаях.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00002_63902995952272_КакВСценарииВыполнитьКодНаЯзыке1С.png)

004. Вместо этого чаще всего лучше создать новый шаг и поместить код на встроенном языке 1С внутрь этого шага.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00003_63902995952679_КакВСценарииВыполнитьКодНаЯзыке1С.png)

005. Теперь, когда мы прояснили этот момент, давай загрузим тестовый пример.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00008_63902995957896_КакВСценарииВыполнитьКодНаЯзыке1С.png)

006. В данном примере показано три варианта выполнить код на стороне менеджера тестирования.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00009_63902995957965_КакВСценарииВыполнитьКодНаЯзыке1С.png)

007. Первый вариант - это выполнить клиентский код.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00012_63902995959577_КакВСценарииВыполнитьКодНаЯзыке1С.png)

008. Второй вариант - это выполнить код на сервере с возможностью обратиться к объекту Vanessa Automation

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00017_63902995964973_КакВСценарииВыполнитьКодНаЯзыке1С.png)

009. Третий вариант - это выполнить код на сервере без возможности обратиться к объекту Vanessa Automation.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00022_63902995970529_КакВСценарииВыполнитьКодНаЯзыке1С.png)

010. Этот вариант является предпочтительным, если нужно просто вызвать серверный код тестируемой конфигурации, при условии, что менеджер тестирования запущен в той же базе, что и клиент тестирования.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00025_63902995974664_КакВСценарииВыполнитьКодНаЯзыке1С.png)

011. Далее рассмотрим вариант, как выполнить код на стороне клиента тестирвания.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00029_63902995978641_КакВСценарииВыполнитьКодНаЯзыке1С.png)

012. Для этого в базе, где запускается клиент тестирования должно быть установлено расширение VA Extension.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00030_63902995978726_КакВСценарииВыполнитьКодНаЯзыке1С.png)

013. Если расширение установлено, то есть два варианта выполнить произвольный код на стороне клиента тестирования.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00031_63902995979002_КакВСценарииВыполнитьКодНаЯзыке1С.png)

014. Первый вариант - это выполнить клиентский код с помощью расширения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00034_63902995980631_КакВСценарииВыполнитьКодНаЯзыке1С.png)

015. Второй вариант - это выполнить серверный код с помощью расширения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава08/images/00039_63902995985958_КакВСценарииВыполнитьКодНаЯзыке1С.png)
