001. Привет! В этом уроке я расскажу тебе про общие настройки формирования отчетов о результате запуске сценария. Давай откроем второй экземпляр Ванессы в режиме обучения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00000_63902990407706_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

002. Эти настройки находятся тут.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00009_63902990420624_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

003. Флаг - добавлять к имени сценария имя сборки. Данная опция бывает полезна, когда одни и теже сценарии запускаются в разном окружении.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00014_63902990426529_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

004. Например могут быть сценарии, которые в CI контуре запускаются в тонком клиенте и в web клиенте.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00017_63902990431000_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

005. Для такого случая можно установить данный флаг.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00019_63902990431831_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

006. В появившемся поле нужно указать уникальной имя сборки.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00022_63902990434451_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

007. При этом для запуска тестов в тонком клиенте надо указать одно имя сборки. А для запуска тестов в web клиенте - другое.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00025_63902990438953_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

008. Таким образом в данном примере будет запущено две сессии тестирования. Одна с первым именем сборки, а вторая - с другим.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00026_63902990439351_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

009. Далее это имя сборки будет добавлено в начале имени каждого сценария, который запускался в данной сессии тестирования.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00027_63902990439757_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

010. Таким образом в таких отчетах как Allure можно будет увидеть запуски одного и того же сценария как разные запуски.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00028_63902990440166_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

011. Движемся далее. Данный флаг включает механизм выгрузки статуса сборки в файл.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00031_63902990441991_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

012. Если флаг установлен - то в данное поле нужно ввести путь к файлу, в который будет выгружен статус сборки.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00039_63902990450747_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)

013. Чтобы узнать подробнее какие бывают статусы сборок перейди по этой гиперссылке.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00042_63902990455257_ЗакладкаСервисОтчетыОЗапускеСценариевОбщие.png)
