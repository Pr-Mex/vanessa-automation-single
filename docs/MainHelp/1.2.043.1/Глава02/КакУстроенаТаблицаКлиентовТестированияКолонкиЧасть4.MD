001. Привет! В этом уроке я расскажу тебе про то, как устроена таблица клиентов тестирования. Давай откроем второй экземпляр Ванессы в режиме обучения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00000_63902991052775_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

002. Посмотрим какие ещё настройки есть в таблице.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00004_63902991061374_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

003. Рассмотрим случай, когда нужно запустить клиент тестирования в web клиенте для файловой базы.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00009_63902991067490_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

004. В этом случае в поле имя компьютера надо также указать значение localhost.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00016_63902991070754_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

005. А в поле порт надо указать значение порта локального web сервера.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00019_63902991075234_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

006. Обычно это 15 38.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00024_63902991078483_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

007. Далее рассмотрим случай, когда нужно запустить клиент тестирования в web клиенте для серверной базы.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00027_63902991082949_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

008. В этом случае в поле имя компьютера надо указать сетевое имя кластера сервера 1С или его ip адрес.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00032_63902991086215_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

009. При этом важно, чтобы менеджер тестирования имел доступ по локальной сети к менеджеру кластера. Иначе подключить клиент тестирования не получится.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00035_63902991090684_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

010. В поле порт при этом надо указать порт кластера серверов 1С.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00036_63902991091090_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

011. Обычно это 15 41.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00041_63902991094355_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

012. Важный момент. Уникальность клиента тестирования для web клиента будет определяться по имени подключения. Другие поля не учитываются.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00046_63902991100463_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

013. Также следует учесть, что при завершении сеанса web клиента нельзя закрывать вкладку браузера или сам браузер.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00052_63902991110099_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

014. В этом случае сеанс web клиента зависнет на сервере.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00053_63902991110169_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

015. Чтобы корректно завершить сеанс web клиента надо нажать в меню: файл. Затем - выход. Как показано на картинке.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00054_63902991110460_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

016. Либо выполнить это же действие другим способом.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00055_63902991110759_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)

017. Тогда сеанс Web клиента завершится корректно.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00056_63902991111039_КакУстроенаТаблицаКлиентовТестированияКолонкиЧасть4.png)
