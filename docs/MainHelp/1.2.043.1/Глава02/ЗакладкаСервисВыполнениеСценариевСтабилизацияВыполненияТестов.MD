001. Привет! В этом уроке я расскажу тебе про то какие настройки влияют на стабильность выполнения сценариев. Давай откроем второй экземпляр Ванессы в режиме обучения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00000_63902988515168_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

002. Эти настройки находятся тут.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00011_63902988529596_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

003. Данное поле устанавливает количество попыток выполнения одного сценария.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00016_63902988535499_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

004. По умолчанию в поле указано значение один.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00019_63902988539969_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

005. Если указано значение более одного, тогда если сценарий не был выполнен успешно, будет выполнено несколько попыток запустить сценарий.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00020_63902988540378_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

006. Продолжим. Этот флаг управляет тем, нужно ли автоматически отключать возможность затирания текста введенного в поле.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00023_63902988542209_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

007. Тут нужно рассказать по подробнее.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00026_63902988546703_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

008. Существует проблема, что при синхронизации состояния формы после завершения серверного вызова, может затираться текст, который в данный момент редактирует пользователь.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00027_63902988547109_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

009. В тэстах это может проявляться так, что тест ввёл значение в поле, но до того как он начал работать с другим полем данное значение было возвращено в предыдущее состояние.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00028_63902988547519_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

010. Данная опция включает механизм, когда свойство "ОбновлениеТекстаРедактирования" у полей формы, будет изменяться после вызова шага Тогда открылось окно

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00031_63902988549350_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

011. Важный момент. Для работы опции в базе клиента тестирования должно быть установлено расширение VAExtension.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00034_63902988553838_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

012. Продолжим. Данное поле устанавливает количество попыток для выполнения действия во время работы сценария.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00037_63902988555683_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

013. Если в поле указано значение больше одного, то это означает, что если действие было выполнено неуспешно, то оно будет выполнено ещё раз через одну секунду.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00040_63902988560147_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

014. Под действием тут подразумевается простое действие с элементом формы такое как установка текста в поле, нажатие на кнопку и так далее.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00041_63902988560556_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

015. При работе тестов в CI контуре рекомендуется увеличивать это значение, чтобы снизить количество ложных срабатываний тестов.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00044_63902988562379_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

016. Продолжим. Это поле управляет тем сколько секунд будет происходить ожидание появления нового окна.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00049_63902988568285_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

017. При работе тестов в CI контуре также рекомендуется увеличивать это значение.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00052_63902988572756_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

018. Продолжим. Это поле управляет тем какой таймаут будет при выполнении асинхронных шагов.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00055_63902988574597_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

019. Асинхронными шагами называются шаги, которые ждут какого-либо события в течение скольки-то секунд.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00058_63902988579089_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

020. При выполнении тестов таймаут таких шагов будет вычислен как максимум данной настройки и значения, которое было передано в шаг.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00059_63902988579503_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

021. При работе тестов в CI контуре также рекомендуется увеличивать это значение.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00062_63902988581340_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

022. Данное поле позволяет задать паузу, которая будет добавлена в шаги, проверяющие открытие нового окна.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00067_63902988587233_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

023. Это может быть полезно, если у форм могут быть дополнительные обработчики инициализации и нужно дождаться их выполнения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00070_63902988591736_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

024. Продолжим. Данная настройка позволяет указать верхнюю границу для длительности выполнения элементарного действия.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00073_63902988593566_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

025. По сути настройка позволяет корректно обработать ситуации, когда клиент тестирования повисает.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00076_63902988597024_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

026. Если задано значение отличное от нуля, то после истечения указанного времени будет вызвано исключение.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00079_63902988598849_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)

027. Если задано значение ноль, то проверка на длительность выполнения действия проводиться не будет.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00084_63902988604750_ЗакладкаСервисВыполнениеСценариевСтабилизацияВыполненияТестов.png)
