001. Привет! В этом уроке я расскажу тебе про основные настройки формирования отчета о запуске сценариев в формате Allure. Давай откроем второй экземпляр Ванессы в режиме обучения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00000_63902989814913_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

002. Эти настройки находятся тут.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00011_63902989829968_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

003. Чтобы после выполнения сценариев сформировался отчет в формате Allure надо установить этот флаг

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00016_63902989835886_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

004. Продолжим. Данный флаг определяет, что в сформированном отчете в шагах вместо имен переменных будут подставлены их значения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00021_63902989841780_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

005. Иногда бывает нужно, чтобы для некоторых переменных не происходила подстановка значений. Эти переменные нужно указать в этом списке.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00026_63902989847663_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

006. В этом поле указывается каталог, в котором будут сформированы файлы отчета.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00031_63902989853552_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

007. Есть альтернативный способ указать каталог отчета.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00034_63902989857028_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

008. Для этого надо заполнить поле каталога, который используется при формировании отчета по нескольким сборкам.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00037_63902989858850_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

009. Это имеет смысл, если установлен флаг добавлять к имени сценария имя сборки.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00044_63902989866370_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

010. Тогда в этом каталоге будет создан ещё один каталог с именем сборки.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00049_63902989872272_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

011. Продолжим. Далее идут три поля, которые позволяют задать три уровня иерархии отчета. Подробно про то, как формируются эти уровни рассказано во внутренней справке, которая вызывается клавишей F1.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00059_63902989878779_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)



![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00064_63902989884724_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

012. В этом поле указывается каталог относительно которого строится иерархия уровней в отчете.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00069_63902989890635_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

013. Данное поле определяют группировку в отчете на закладке сьютс.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00074_63902989896543_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

014. В этом поле указываются тэги. Если у сценария будет найден такой тэг, то он не будет выполняться и его статус в отчете будет - пропущен.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00079_63902989902467_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)

015. На этом всё, переходи к следующему уроку интерактивной справки.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00082_63902989906953_ЗакладкаСервисОтчетыОЗапускеСценариевAllureОсновные.png)
