001. Привет! В этом уроке я расскажу тебе про настройки скорости выполнения сценариев. Давай откроем второй экземпляр Ванессы в режиме обучения.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00000_63902988188865_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)

002. Эти настройки находятся тут.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00009_63902988201742_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)

003. Данный флаг включает асинхронный режим выполнения шагов.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00014_63902988207634_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)

004. Если флаг установлен, то каждый следующий шаг будет выполняться после паузы.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00019_63902988213736_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)

005. В данном поле нужно указать значение паузы между шагами. Минимальное значение - одна десятая секунды.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00022_63902988216369_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)

006. Обычно данная опция используется для искусственного замедления выполнения сценария. Например, при демонстрации работы сценария другому пользователю.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00025_63902988220866_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)

007. При обычной работе и при запуске тестов в CI контуре рекомендуется этот флаг снимать.

![](https://vanessa-files.do.bit-erp.ru/Doc/1.2.043.1/MD/Глава02/images/00028_63902988222699_ЗакладкаСервисВыполнениеСкоростьВыполнения.png)
